# サービス解説 07: [付録] 基本のSQLチートシート

Inception課題において、データベース（MariaDB）の内部を直接操作するのは主に「初期化（データベースとユーザーの作成）」の時だけですが、トラブルシューティングや確認作業のために基本的なSQL文法を知っておくと非常に役立ちます。

このチートシートでは、コンテナ内（またはVM上）でデータベースの状態を確認・操作するための必須コマンドをまとめました。

---

## 1. コンソールへのログインと基本操作

Linuxのシェルから MariaDB コンソール空間に入る（および出る）ためのコマンドです。

| 目的 | コマンド（Linuxのターミナルで実行） |
| :--- | :--- |
| **rootでログイン** | `mysql -u root -p` |
| **特定のユーザーでログイン** | `mysql -u wp_user -p` (パスワードを求められます) |

MariaDBコンソールに入った後（`MariaDB [(none)]>` というプロンプト）の基本操作です。

| 目的 | SQLコマンド（末尾の `;` を忘れないこと） |
| :--- | :--- |
| **コンソールから抜ける** | `exit;` または `quit;` |
| **打っている途中のコマンドをキャンセル** | `\c` (セミコロンを忘れて改行してしまった時などに便利です) |

---

## 2. データベースとテーブルの確認 (表示・選択)

自分が作ったデータベースや、WordPressが自動生成したデータの中身を「見る」ためのコマンドです。

| 目的 | SQLコマンド |
| :--- | :--- |
| **データベース一覧を表示** | `SHOW DATABASES;` |
| **使うデータベースを選択する** | `USE wordpress_db;` (※これを打たないとテーブルの中身が見られません) |
| **テーブル一覧を表示** | `SHOW TABLES;` (※ `USE` でDBを選択した後に実行します) |
| **特定のテーブルの構造を見る** | `DESCRIBE wp_users;` (ユーザー情報を格納するテーブルの構造を見ます) |

---

## 3. アカウント管理 (作成・権限付与)

初期化スクリプト（`init.sh` 等）の中で必ず使用する、データベースの根幹設定に関わるコマンド群です。

| 目的 | SQLコマンド |
| :--- | :--- |
| **データベースの作成** | `CREATE DATABASE wordpress_db;` |
| **ユーザーの作成とパスワード設定** | `CREATE USER 'wp_user'@'%' IDENTIFIED BY 'password123';`<br>(※ `'@'%'` は「どのIPアドレスからでも接続OK」を意味します) |
| **ユーザーに全権限を付与** | `GRANT ALL PRIVILEGES ON wordpress_db.* TO 'wp_user'@'%';`<br>(※ `.*` は「そのDBの全テーブル」を意味します) |
| **権限の変更を確定させる** | `FLUSH PRIVILEGES;` |
| **登録されているユーザー一覧を見る** | `SELECT user, host FROM mysql.user;` |

---

## 4. データの中身を見る (SELECT文)

WordPressが正常にインストールされたか、どんなデータが入っているかを確認するためのコマンドです。
*(※ 事前に `USE wordpress_db;` を実行しておく必要があります)*

| 目的 | SQLコマンド |
| :--- | :--- |
| **テーブル内の全データを表示** | `SELECT * FROM wp_users;` (登録されているユーザー一覧が見えます) |
| **特定の列だけを表示** | `SELECT user_login, user_email FROM wp_users;` |
| **条件で絞り込んで表示** | `SELECT * FROM wp_users WHERE user_login = 'admin';` |

---

## 💡 トラブルシューティングのコツ
もしWordPressコンテナが「データベース接続確立エラー (Error establishing a database connection)」を出して画面が表示されない場合、以下の手順で原因を切り分けられます。

1. `docker exec -it mariadb bash` でDBコンテナに入る。
2. `mysql -u [WordPressに渡したユーザー名] -p` でログインを試みる（これで弾かれたらパスワードが間違っているか、ユーザー作成が失敗しています）。
3. ログインできたら `SHOW DATABASES;` を打つ（ここで `wordpress_db` が見えなければ、権限付与(`GRANT`)が失敗しています）。
